# ByteBites Tinker Activity

The core concept for this tinker is object-oriented design: defining classes with attributes and methods, and understanding how objects compose and relate to one another. Students need to see that classes like Customer, Order, and FoodItem aren't isolated. They are connected. A Customer holds multiple Orders, and each Order holds multiple FoodItems. The biggest struggle point is likely to be reasoning about these class relationships. Copilot was helpful for scaffolding drafts of class structure and generating comprehensive test cases, but it can be misleading by producing such polished, complete code that students accept it without understanding the reasoning why the design works this way. Rather than showing a student the answer, I could ask them to trace through what happens step by step when a customer places an order: "Which objects are created? Where does each one get stored? How do they connect?" This forces them to trace the chain themselves.
